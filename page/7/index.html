第一章 基础知识

发表于 2018-03-26 | 分类于 汇编

CPU

8086/8088CPU组成

8086/8088CPU共同点

​ 都是16位CPU. 运算器一次最多可以处理16位数据, 寄存器最大宽度为16位, CPU内部数据总线为16位

8086/8088CPU区别

CPU 指令队列容量 每个总线周期从内存中读取放入指令队列 外部数据总线宽带
8086 6个字节 2个字节 16位
8088 4个字节 1个字节 8位
  • 总线周期: BIU完成一次访问存储器或端口所需要的时间

字word

​ 计算机内部一次可以处理的二进制位数. 8086CPU中寄存器位16位, 所以字为16位

内存中字的存储

​ 高地址存放高位字节, 低地址存放低位字节

0 1 2 3
20H 4EH 12H 00H

从地址为1的内存单元中取出1个字的数据, 为124EH

寄存器(14个16位寄存器)

段寄存器 作用
代码段寄存器CS 指向代码段
数据段寄存器DS 指向数据段
堆栈段寄存器SS 指向堆栈段
附加段寄存器ES 指向附加段
  • 标志寄存器FR: 又称为状态寄存器PSW

数据寄存器

​ 为了兼容上一代的8位CPU, 数据寄存器可分为高位寄存器和低位寄存器

低位寄存器 高位寄存器
AX AL AH
BX BL BH
CX CL CH
DX DL DH

简单汇编指令

汇编指令 操作 描述
mov ax, 18 将18送入AX AX = 18
mov ah, 78 将78送入AH AH = 78
add/sub ax, 18 AX中的数据加/减上18 AX = AX +/- 18
add/sub ah, 18 AH中的数据加/减上18 AH = AH +/- 18

  • 汇编指令不区分大小写: mov ax, 18 <=> MOV AX, 18

    ​

  • 汇编指令中操作数的位数应该一致: mov ah, bx是错误的, ah是8位寄存器, bx是16位寄存器

物理地址

​ 8086CPU中地址总线是20位的, 但存储地址的寄存器是16位的. 所以将两个16位地址合成一个20位地址

  • 物理地址 = 段地址 16 + 偏移地址 = 段地址 10H + 偏移地址
  • 逻辑地址 = 段地址 : 偏移地址

寄存器CS和IP

​ CS * 16 + IP指向当前要读取的指令的地址

程序执行

1. 从CS : IP指向的内存单元读取指令并放入指令缓存器
 2. IP = IP + 所读指令的长度, 从而指向下一条指令(IP永远指向指令, 因此冯诺依曼体系中主要根据IP指向内容判断是内存中的内容是指令还是数据)
  3. 执行指令

DeBug程序

打开debug

  1. 打开DOSBOX

    ​

  2. 输入下列指令

指令

R

  • r, 查看所有寄存器的内容
  • r + 寄存器名, 在提示的: 后输入值, 修改指定寄存器的值

D

  • d 段地址 : 偏移地址, 查看此物理地址开始的128个内存单元的内容

  • d 段地址 : 起始偏移地址 结束偏移地址, 查看指定范围的内存单元

  • 右边的值表示每个内存单元的值对应的ASCII字符, 若ASCII字符不能显示就用’.’代替

E

  • e 段地址 : 偏移地址 数据 数据 数据 …., 来修改从指定内存开始的内存单元
  • e 段地址 : 偏移地址 回车。 然后在.后面输入打算替换的数据, 改完后按空格修改下一个, 按回车结束

U

  • u 段地址 : 偏移地址, 产看指定地址开始的汇编指令

T

  • t , 执行CS : IP的指令

A

  • a 段地址 : 偏移地址, 向指定内存地址写入汇编码

汇编指令转为机器码

  • 当机器码表示为针对第一个操作数时, d = 1; 若针对第二个操作数, d = 0
  • 字操作, w = 1; 字节操作, w = 0
  • disp表示位移量, 比如mov ax, [si] + 18, 则disp为18
  • reg表

第一章 计算机系统组织

发表于 2018-03-25 | 分类于 操作系统

计算机系统操作

启动计算机

当打开电源或重启时, 计算机运行BIOS, 它将定位操作系统的内核并将其装入内存中, 然后操作系统执行第一个进程如init, 并等待事件的发生

  • BIOS: 固化在ROM中的引导程序, 保存着基本的输入输出程序

  • 内核core: 操作系统最核心的部分, 它提供了程序对计算机硬件的安全访问

  • 外壳shell: 用于控制操作系统的程序, 它提供了与用户交互的界面

    ​

中断

指停止当前程序, 并开始处理新程序, 直至完成后继续被暂停的程序. 通过中断产生了新的事件, 位模式的切换也依赖于中断

事件

通过软件、硬件的中断产生

  • 软中断: 软件通过执行特定的操作产生, 比如通过trap产生系统调用

  • 硬中断: 硬件通过总线发送中断信号产生

    ​

中断向量表

用一个表保存所有程序的地址和状态, 然后将表保存到内存中从0开始的地址. 产生中断时, 根据中断向量根据中断码在中断向量表中找到对应的中断处理程序的入口地址, 然后跳转到对应的中断处理程序处理中断

I/O结构

I/O操作

控制器先检查指令寄存器的内容决定要采取的操作, 然后将数据寄存器中的内容传输到本地缓冲区. 传输完成后, 控制器会产生一个中断信号通知驱动程序传输完成

  • 驱动程序: 存在于设备控制器上, 它提供了硬件与软件系统的接口, 充当两者的桥梁
  • 缺点: 因为数据寄存器只能保存一个字节, 所以只适用于少量数据的传输

DMA直接内存访问

一大块数据的传输, 只产生一个中断来告知驱动程序已经完成

  • 原因: 对于大块数据的传输, 若采用多次I/O中断驱动的方法传输, 会产生超载问题

    ​

处理器

  • 多核处理器: 将多个计算机计算机引擎设计到单个芯片上

    ​

操作系统结构

多道程序设计

操作系统同时将多个任务保存在内存中, 当一个任务执行完后立即执行另一个任务

分时系统

多用户共享计算机, 通过在作业之间快速进行切换让用户感到系统只为自己所用

操作系统操作

CPU操作模式

实模式和保护模式:

​ 80286CPU及以前的CPU, 内存地址只有16位, 随着可访问内存增多. 为了兼容, 将旧的部分设为实模式, 新增部分设为保护模式

  • 实模式: 所有内存都是可读写的
  • 保护模式: 一. 提供保护机制, 不是所有段都是可读写的; 二. 可以访问的内存空间变大

用户模式和系统模式:

​ 通过在硬件中添加一个称为模式为的位, 来区分用户定义的代码和操作系统代码

  • 用户模式: 只能执行非特权指令. 若执行特权指令, 会通过陷阱的形式中断

  • 系统模式/特权模式/内核模式: 能够执行全部指令, 包括特权指令

    ​

定时器

设置在给定时间后中断计算机

  • 原因: 为了防止用户程序陷入死循环或不进行系统调用, 导致不将控制权返回到操作系统

存储管理

高速缓存的一致性

对于存储在某个高速缓存中的值A, 当它的值改变. 所有保存了A的副本的高速缓存都要改变它们存储的A值
