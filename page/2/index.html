第四章 线程

发表于 2018-05-10 | 分类于 操作系统

线程

  • 共享堆内存和全局变量

线程模型

  • 根据是否受到内核的干预分为: 用户线程和内核线程
线程模型 含义 优点 缺点
多对一 多个用户线程映射一个内核线程 内核线程少, 减少开销 只有一个内核线程, 所以用户线程不能并发执行
一对一 一个用户线程映射一个内核线程 多个用户线程能并发执行 内核线程多, 开销大
多对多 多个用户线程映射多个内核线程 多个用户线程能并发执行. 且内核线程数量相对固定, 开销不大

线程取消

​ 在线程完成前终止线程

  • 异步取消: 立即终止目标线程
  • 延迟取消: 不断检查目标线程是否应该取消

线程池

​ 在进程开始时创建一定数量的线程, 并放入到池中等待工作. 当服务器收到请求时, 唤醒池中的一个线程并传递给要请求. 当请求完成, 将线程放回线程池

第四章 处理器

发表于 2018-04-16 | 分类于 计算机组成与原理

寻址方式

立即数寻址

​ 地址 = 指令立即数

寄存器寻址(R型指令)

​ 地址 = 指令rs

基址寻址(i型指令)

​ 地址 = 指令address + 基址寄存器(比如ds, es)

PC相对寻址(分支指令)

​ 地址 = (指令地址符号扩展为32位, 再 << 2位, 得到字节地址) + PC寄存器

伪直接寻址(J型指令)

​ 地址 = PC寄存器高4位 : (指令地址低26位字地址 << 2位, 得到字节地址)

组合单元, 状态单元

​ 根据是否含有存储功能, 将元件分为组合单元和状态单元

  • 组合单元: 没有存储功能
  • 状态单元: 有存储功能

有效, 无效

  • 有效: 逻辑真
  • 无效: 逻辑假

数据通路单元

​ 除特殊说明外, 连接通路单元的数据线宽度为32位

寄存器堆

​ 包含一系列寄存器, 通过提供寄存器号进行读写

ALU

符号扩展

​ 将16位地址地址扩展为32位, 新的高16位为符号位

数据存储单元

​ 向内存中的存储单元读取或写入数据(比如sw, lw)

ALU控制信号

​ 控制ALU执行不同的功能, 由指令的func字段和ALUOp共同生成(详见P174页4-12, 4-13)

指令 ALUOp ALU控制信号

控制信号

​ 增加7个1位控制信号, 用于控制各个数据通路单元

控制信号名

​ 除PCSrc外, 都可以通过指令的op字段获得. PCSrc可以通过将Branch与ALU的零标志得到(前者表示为分支指令, 后者表示源操作数相等)

控制信号名 无效时(电平=0) 有效时(电平=1)
RegDist 写寄存器的目标寄存器号来自rt字段 写寄存器的目标寄存器号来自rd字段
RegWrite 无 向寄存器写数据
ALUSrc ALU的第二个操作数来自寄存器 ALU的第二个操作数为指令的低16位符号扩展
PCSrc PC = PC + 4 PC = 目标地址
MemRead 无 从存储单元读数据
MemWrite 无 向存储单元写数据
MemtoReg 写入寄存器的数据来自ALU 写入寄存器的数据来自存储器
